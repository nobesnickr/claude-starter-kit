# Your first session (about 15 minutes)

Welcome. This is a guided first session, not a manual. Do these in order; ignore everything else for now.

## Before you start
You should have already: installed Claude Code, signed in, and run `/setup-me`. If you haven't, see the [README](../README.md) first. (It also explains the difference between the **Terminal** and **Claude Code** — worth a 20-second read if those words are new.)

## 1. Read one short page (3 min)
Open [AI-LITERACY.md](AI-LITERACY.md) and read it. It explains the one habit — **Cite & Check** — that keeps your work accurate. Everything else here builds on it.

## 2. Start a project (2 min)
Open the **Terminal** app, type `claude`, and press Enter. Once Claude Code is running, just ask it:
> "Start a new project for my weekly report using the starter template, and tell me the goal it should record."

It creates the folder and copies the template in for you — you don't find or copy any files yourself. Tell it the project's goal in one sentence so it can fill in the README.

## 3. Bring in real data (3 min)
Take the spreadsheet you actually use for reporting. In Google Sheets: **File → Download → Comma-separated values (.csv)** — it saves to your Downloads. Then tell Claude Code:
> "I just downloaded the CSV to my Downloads — move it into this project."

It finds the file and places it for you. This is the single most important step — from here on, Claude works from your *real numbers*, not its imagination.

## 4. Practice Cite & Check once, guided (5 min)
Now ask Claude Code:
> `/datastrat` then: "Read my CSV and tell me the three headline numbers for this week. Cite the exact cell for each."

When it answers:
- **Cite:** confirm each number names a cell.
- **Check one:** open the CSV (double-click it to open in Numbers or Excel) and look at *one* of those cells yourself. Does it match?
- If it matches, you've just done the whole habit. If it doesn't, you've just caught a mistake before it reached a report — which is exactly the point.

## 5. You're done for today
That's the core loop: real data in, cited answers out, spot-check one. When you want a thinking partner for *what to do* rather than *what the data says*, try `/bizstrat`. See [CONSULTANTS.md](CONSULTANTS.md) for both.

The kit will grow — more consultants and daily routines are planned for future versions. For now, this is everything; don't go looking for the rest yet.
